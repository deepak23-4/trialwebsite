Combinator selector





They are called combinator selectors because they combine other selectors in a way that gives them a useful relationship to each other and the location of content in the document.

There are four different types of combinator selectors, they are:

  1. Descendant selector
  2. Child selector
  3. Adjacent sibling selector
  4. General sibling selector


descendant selector
It is represented by a single space ( ) between two selectors without a combinator
Descendant means anywhere nested within the web document in a DOM tree, it could be a direct child, could be five levels deep, its still a descendant.
We can combine two or more than 2 tags from generic to specific
Example: 
          ul li{color:red;}

          it means "any list item that has a descendant of any unordered list will have a font color red" and it will have a font color "red" for all the list items under unordered list.


child selector
A child selector is represented by greater than sign (>) which is placed in between two CSS selectors
It targets only those elements which are the direct child of its parents element in other words it targets those elements which are direct descendants only
The only different in between descendant selector ol li{ } and child selector ol>li{ } is that in descendant selector it will target the direct child, child's child and it will go further down with in the same hierarchy to all its descendants however in child selector it will only target the child selector which is the direct descendant.
Exammple: 
          ul>li{
            color:blue;
          }

          it means as direct child if it has 5 list items so all will be selected however if its nested list with in the unordered list so it will not select the further down level.


Adjacent sibling selector
The adjacent sibling combinator selector is represented by (+) character sign and it allows you to target an element that is directly after the specific element.
adjacent sibling selectors can only be applied to the elements of the same parent

Example: 
          img + p {
            font-weight:bold}
          }
          it means here the paragraph which comes directly after img tag will have bold fonts weight.


general sibling selector
general sibling selector is represented by (~) character The general sibling selector is very similar to the adjacent sibling selector, the difference is that that the element being selected doesn't need to immediately succeed the first element, it can appear anywhere after it within the same parent element.
Syntax: 
          p~p{color:blue; }
